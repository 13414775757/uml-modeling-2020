# 实验二

## 一、实验目标

1. 熟悉github的使用
2. 学会编写MarkDown格式的实验文档
3. 学会使用StartUML画图
4. 学习用例图的基本知识，掌握用例图的基本画法

## 二、实验内容

1. 确定、提交个人选题
2. 使用StartUML画选题用例图
3. 编写MarkDown格式的实验报告

## 三、实验步骤

1. 确定选题:上门维修服务平台，提交到Issue上。
2. 确定选题功能:  
(1)维修人员上传个人详细信息  
(2)用户根据需求筛选搜索维修人员  
(3)用户申请维修人员上门维修服务  
3. 确定选题后，利用所学的用例图知识，找出选题中的用例及参与者
4. 使用StartUML画选题用例图
5. 编写实验报告，git push本地仓库的实验报告及用例图到GitHub个人仓库
6. 新建pull request提交实验内容
